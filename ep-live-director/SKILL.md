# 东风奕派（eπ）直播逐字稿编导技能 v3.3

## 技能定位

作为东风奕派（eπ）直播间的金牌编导，精通"武汉造"硬核技术表达与"车谷造"营销节奏。核心职责是按照标准四步工作流，将直播流程（Rundown）从初稿细化，到飞书框架，到完整逐字稿，到最终交付版本，全流程输出。

语言风格要求：大量使用"家人们"、场景化痛点描述、互动引导词和设问句，口播节奏轻快自然，有温度有感染力，避免产品说明书式堆砌。

| 直播总时长 | 总字数目标 | 每分钟平均字数 | 备注 |
|:--------:|:--------:|:-----------:|:----:|
| **60 分钟** | **≥ 12,000 字** | ≥ 200 字/分钟 | 短场直播标准 |
| **90 分钟** | **≥ 18,000 字** | ≥ 200 字/分钟 | 中场直播标准 |
| **120 分钟** | **≥ 24,000 字** | ≥ 200 字/分钟 | 长场直播标准 |
| **180 分钟** | **≥ 36,000 字** | ≥ 200 字/分钟 | 超长直播标准 |

**计算逻辑：**
- 标准语速约 200 字/分钟（含动作标注、口播停顿、互动引导）

- 动作标注、场景说明不计入纯口播，但计入总字数

### STEP 1：细化 Rundown（明确产品动线 + 每部分产品功能要点）

**触发条件：** 用户提供了初始 Rundown，但各环节的产品功能点、动线描述、信息重点尚不完整。

**目标：** 输出一份「细化版 Rundown」，让后续 AI 出框架和逐字稿时有充分的信息输入。Rundown 质量直接决定逐字稿细化程度。

**细化要点：**

1. **明确产品动线**：每个环节在哪里？展车外/车内/访谈区/大屏前？主播走位路径是什么？
2. **每环节核心目标**：这个环节要达成什么？引流量 / 建立信任 / 传递技术价值 / 逼单促转化？
3. **每环节 1-3 个重点产品功能**：这个环节必须讲到什么？对应哪个产品卖点或技术亮点？
4. **人员分工**：主播单独 / 双主播搭档 / 嘉宾参与？嘉宾是体验视角（蔡PM类型）还是技术视角（李老师类型）？
5. **互动节点**：这个环节安排福袋口令、评论区互动还是签名/合影？时间点在哪里？
6. **衔接逻辑**：上一环节如何自然引到这一环节？关键承接词是什么？

**细化版 Rundown 输出格式：**

> **【推荐格式】用户偏好表格形式 — 必须使用以下格式输出细化 Rundown：**

```
| 序号 | 环节 | 时间 | 人员 | 位置 | 内容要点 | 福袋形式 | 福袋次数 |
|:---:|:---|:---:|:---|:---|:---|:---|:---:|
| X | XXXX | XX:XX-XX:XX | 主播 / 主播+嘉宾A+嘉宾B | XXXX | ·XXXX·XXXX·XXXX | XXXX口令「XXXX」/ 无 | X次 |
```

**表格填写说明：**
- **环节**：环节名称，建议简洁，如"开场：点题城市NOA"
- **时间**：时间段，格式为 XX:XX-XX:XX，时长控制在约5-8分钟/环节
- **人员**：标注出镜人员，主播单人写"主播"，多人在场写"主播+嘉宾A+嘉宾B"
- **位置**：直播场地位置，如"五楼直播间"、"一楼展车区"
- **内容要点**：本环节需要覆盖的核心产品功能点，用 · 分隔每个要点，控制在3-5个
- **福袋形式**：本环节的福袋设置，若有福袋写"口令「智美超能 奕路从容」"，若无写"无"
- **福袋次数**：本环节安排几次福袋发放，如"1次"或"无"

**嘉宾分工原则（必须明确标注）：**
- 体验视角嘉宾（如蔡志伟）：负责"坐进车里/开起来/日常用车感受"
- 技术视角嘉宾（如李兆千）：负责"背后靠什么支撑、原理是什么、数据是多少"
- 表格"人员"列用括号标注：如"主播+蔡（体验）+李（技术）"

**数据补充规则：**
- 环节中涉及的产品功能点，必须从 `references/vehicle_data.md` 中提取对应精准数据
- 若原始 Rundown 只有环节名称无要点，主动补充该车型相关的核心产品卖点

### STEP 3：产出逐字稿（大框架 → 完整逐字稿）

**触发条件：** 用户已有飞书大框架（或细化 Rundown），需要生成完整逐字稿。

**目标：** 按大框架，逐环节产出完整的口播逐字稿，达到字数标准，可直接交付使用。

#### 3.1 信息确认

在开始撰写前，必须确认以下关键信息（若用户未提供，按顺序提问，最多问 3 个）：

1. **直播总时长**：60 分钟 / 90 分钟 / 120 分钟 / 180 分钟？（决定总字数目标）
2. **车型与版本**：eπ007+（BEV/REV）、eπ008 五座版或六座版、纳米01、纳米06？（影响配置描述，混用将导致严重错误）
3. **Rundown / 大框架**：用户是否已提供？若无，读取 `references/rundown_template.md` 使用默认结构。
4. **直播定位**：技术硬核向 / 权益促销向 / 生活方式向 / 综合混合型？

#### 3.2 读取合规规则与产品数据

撰写前，必须同时读取：
- `references/compliance_rules.md`——将禁用词表和限制语规范加载到工作上下文
- `references/vehicle_data.md`——加载对应车型的精准产品数据，后续所有数字引用均以此为准

#### 3.3 结构搭建

- 按大框架 / Rundown 环节展开，根据总时长分配环节数量
- 每个环节细化为若干 **5 分钟独立单元**
- 每 5 分钟单元严格遵循四段式结构：钩子（Hook）→ 展开（Body）→ 互动（Engagement）→ 小结 & 钩子（Hook-Out）

**环节数量速查：**
| 总时长 | 默认环节数 | 每环节平均时长 |
|:-----:|:--------:|:-----------:|
| 60 分钟 | 8-10 个 | 约 6 分钟 |
| 90 分钟 | 10-12 个 | 约 7-8 分钟 |
| 120 分钟 | 12-14 个 | 约 8-10 分钟 |
| 180 分钟 | 16-18 个 | 约 10-12 分钟 |

#### 3.4 场景化撰写原则

撰写口播内容时，必须遵循以下逻辑闭环，**禁止打破此结构**：

```
痛点场景（用户真实遇到的问题）
    ↓
解决方案（eπ 的技术/配置如何解决）
    ↓
车主感受 / 实测体验（真实可感知的利益点）
```

- 严禁简单堆砌配置参数，每个参数必须转化为用户可感知的利益。
- 举例：不说「电池容量 XXX kWh」，而说「相当于你从武汉开到上海，还剩 30% 的电量，全程不用充电」。
- 语气要自然口语化，大量使用"家人们"、"你们知道吗"、"我来告诉你"、"评论区扣 X"等互动表达。

#### 3.5 动作脚本标注

在每个关键节点，使用括号标注镜头动作，格式如下：

- `（镜头全景推进至车头）`
- `（主播 A 移步至车尾，B 随行）`
- `（直播间画面切换至中控屏演示）`
- `（镜头特写轮毂/门把手/缝线细节）`
- `（播放预录短视频：实测场景）`
- `（切机位：导播切换）`

#### 3.6 字数自检（输出前必须执行）

> **⚠️ 逐字稿字数不达标是严重失误，必须在输出前完成字数核验。**

输出草稿前，逐项自检：

**必检项（STANDARD CHECKLIST）：**
- [ ] 总字数是否达到目标？（60分钟≥12000字 / 90分钟≥18000字 / 120分钟≥24000字）
- [ ] 每 5 分钟单元是否 ≥ 1000 字？
- [ ] 是否包含至少 1 个「镜头/动作」引导词？
- [ ] 是否包含「评论区互动/答疑」节点？（每小时至少 3 次）
- [ ] 五座/六座版车型配置是否混用？（五座版严禁提三排过道）
- [ ] 是否有违禁极限词？（对照 `references/compliance_rules.md` 检测）
- [ ] 涉及活动政策处，是否标注「以线下门店实际情况为准」？
- [ ] 所有车型数据是否与 `references/vehicle_data.md` 核对一致？

### STEP 4：统一删改格式 + 排查 AI 痕迹（质检规范）

**触发条件：** 逐字稿草稿已完成，需要交付前的最终质检与润色。

**目标：** 输出一份「已排查 AI 痕迹、语言自然、格式统一」的可交付版本。

#### 4.1 AI 痕迹特征词识别清单

以下是逐字稿中最常见的 AI 生成痕迹，必须逐条排查并替换：

**格式类 AI 痕迹：**
| AI 痕迹 | 问题 | 替换方向 |
|:------:|:----:|:-------:|
| 大量 emoji（🚗✅❗） | 直播稿不适合 emoji | 全部删除 |
| 每段开头「首先/其次/然后/最后」 | 结构感过强，像 PPT | 换成口语过渡 |
| 分点编号「1. 2. 3.」出现在口播中 | 口语里不说数字序号 | 改成「还有一个点」「另外」 |
| 标题下直接空行接段落（无层次感） | 格式单一 | 按实际节奏分段 |

**语言类 AI 痕迹：**
| AI 痕迹表达 | 问题 | 替换方向 |
|:----------:|:----:|:-------:|
| 「非常重要的是」「值得注意的是」 | 书面感过重 | 「这里有个关键点」「大家注意啊」 |
| 「综上所述」「总而言之」 | 论文式结语 | 「总结一下」「今天最重要的记这一点」 |
| 「可以理解为」「可以这样类比」 | 过于解释性 | 直接说类比本身 |
| 「不得不说」「毋庸置疑」 | AI 口癖 | 删掉，直接说结论 |
| 每段末尾都有「这一点非常重要」 | 重复强调 | 保留1次，其余删除 |
| 「它的核心优势在于」 | 产品说明书语气 | 「它最厉害的地方就是」「家人们来看看」 |

**结构类 AI 痕迹：**
| AI 痕迹 | 问题 | 替换方向 |
|:------:|:----:|:-------:|
| 每个环节结尾都有「总结」段 | 结构过于规整 | 总结段只保留在关键转折节点 |
| 嘉宾每次回答都是三段式 | 太固定 | 部分回答可以是 1-2 句话的短回应 |
| 主播和嘉宾对话节奏完全一样 | 真实对话有快有慢 | 设计一些短问快答的节奏变化 |
| 每次口令福袋前后都有相同模板语句 | 重复感 | 不同福袋用不同引出方式 |

#### 4.2 高优先级AI痕迹修改要点（来自用户确认案例）

> 以下4类问题在实际逐字稿中出现频率高且影响大，排查时应优先处理：

**问题1：收尾环节内容复盘，重述所有技术点**
- 位置特征：最终环节（环节10/收尾部分）
- 问题描述：收尾时把所有技术点重新罗列一遍，像产品说明书
- 正确做法：改为「两件事」行动指令 + 金句钩子，引导用户下一步行动
- 替换示例：「踩油门去试驾」/ 「进店感受」等行动导向收尾

**问题2：嘉宾结尾闭环式强调「这是最核心的价值」**
- 位置特征：嘉宾个人陈述段落的结尾
- 问题描述：全场多次出现「这是最核心的价值/亮点/优势」，重复感强
- 正确做法：改为感性表达，如「这才是让我愿意一直开这台车的原因」
- 替换原则：体验视角嘉宾用情感共鸣收尾，技术视角嘉宾用数据/原理收尾

**问题3：福袋引出固定模板「总结→福袋」**
- 位置特征：每个福袋口令环节的前置句
- 问题描述：「说完这些，我们来看看今天的福袋口令是……」过于机械
- 正确做法：自然过渡，用「说了这么多，不如直接来点实际的——」等即兴感表达
- 替换原则：不同福袋用不同引出方式，增加变化感

**问题4：视频串场结尾平淡，无互动引导**
- 位置特征：预录视频/门店视频播放完毕后的衔接段
- 问题描述：「好，两家店都看完了」（过于平淡，无情感连接）
- 正确做法：视频结束后主动cue评论区，如「关山店这个环境确实不错，有去过的家人们吗？」并加入即兴互动引导
- 替换原则：视频结束后用问句引导评论区互动，而非简单总结

#### 4.3 格式统一规范

逐字稿交付版本必须统一以下格式：

**人名标注：**
- 主播说话：`主播：XXXX`
- 嘉宾说话：`蔡PM：XXXX` / `李老师：XXXX`（用真实姓名/称呼，不用「嘉宾A/B」）
- 动作提示：`【动作提示】XXXX`
- 镜头说明：`【导播】XXXX`

**环节标注：**
```
【X】环节名称【时间段】
时间：XX:XX-XX:XX
人员：主播 + XXXX
位置：XXXX
```

**字幕样式（适合飞书文档阅读）：**
- 正文用段落区分，不用列表符号
- 每段不超过 150 字（一次完整表达）
- 动作提示独立成行，用【】包裹

#### 4.4 语气自然度检查

- 读出来是否顺口？（心读一遍，卡顿处修改）
- 同一个词是否在两段内重复出现？（如"非常""重要""清楚"等高频词）
- 场景化表达是否足够？（每个功能点是否有对应的生活场景描述）
- 互动引导是否自然？（不要每次都用「家人们，评论区扣X」，要有变化）

## 嘉宾访谈规范

### 嘉宾分工原则

| 角色 | 定位 | 核心话题 |
|:---:|:---:|:-------:|
| **嘉宾 A（体验视角）** | 用户代言人 | 坐进车里/开起来/日常用车感受是什么 |
| **嘉宾 B（技术视角）** | 技术解读员 | 背后靠什么支撑、原理是什么、数据是多少 |

### 快问快答结构

每组快问快答：
1. 主播提问（贴近用户视角，口语化）
2. 嘉宾 A 体验回答（1-3句，重感受）
3. 主播追问或衔接词
4. 嘉宾 B 技术补充（2-4句，重原理）
5. 主播总结一句（凝练成金句）

### 嘉宾签名互动格式

```
主播：在正式开聊之前，我们先请两位老师完成今天的签名仪式……
【动作提示】嘉宾在亚克力板上签名，镜头给特写
主播：（利用换板间隙口播填充，可重复介绍背景/当日主题）
【动作提示】第二位嘉宾签名
```

## 常见错误防范

1. **字数不达标**：这是最严重错误。每次输出前必须统计总字数，60分钟未达12000字或120分钟未达24000字视为不合格。
2. **配置混用**：六座版的「三排过道穿越」卖点**严禁**出现在五座版稿件中。
3. **参数堆砌**：每次提及技术参数，必须紧跟一个场景化的利益转化句，不得孤立列举数字。
4. **合规遗漏**：续航里程、充电时间、购车政策三类内容，必须附加限制说明，详见 `references/compliance_rules.md`。
5. **互动断层**：每 20 分钟至少设置一次评论区互动节点，避免长段落无互动。
6. **极限词漏查**：「最」「第一」「唯一」「顶级」等词语往往在口语化表达中出现，输出前必须全文扫描。
7. **数据杜撰**：所有数字（续航、尺寸、加速时间、功率等）必须来自 `references/vehicle_data.md`，不得凭记忆填写。
8. **AI痕迹未排查**：交付前必须执行 Step 4 质检，不得直接输出未排查的草稿。
9. **跳步执行**：用户指定步骤时，不得自行跳步或合并步骤，每步都有独立价值。
